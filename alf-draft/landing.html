PIPELINE

상담 데이터부터 ALF 등록까지, 끊김 없는 자동화

채널톡 Open API로 메시지를 수집하고, OpenAI로 분석·작성하고, Supabase에 검수 가능한 초안 형태로 적재합니다.

01SOURCE
수집
Channel Talk Open API v5에서 상담방·메시지·태그·매니저 정보를 페이지네이션으로 수집합니다. 매일 새벽 GitHub Actions로 증분 수집.
Channel Talk API 5개 state 증분 수집
02STORE
구조화 저장
시간·매니저·내부메모(private)·역할까지 메시지 단위 메타를 JSONB로 정규화해 Supabase에 적재. 검색·필터·통계 모두 SQL로 가능합니다.
Supabase JSONB KST timestamp private flag
03SYNTHESIZE
생성·검수
OpenAI gpt-4o-mini가 채널톡 ALF 가이드라인(## 문제 상황 + ## 해결 방법)에 맞춘 아티클·FAQ·규칙 초안을 작성. 검증기가 형식 오류를 자동 보정합니다.
gpt-4o-mini ALF 가이드라인 verify_draft
CAPABILITIES

검수자의 시간을 아끼는 기능들

단순 LLM 호출이 아니라, ALF 채널톡 가이드라인을 깊이 반영한 워크플로우.

🧩가이드라인 내장 프롬프트

채널톡 ALF 팀 공식 가이드(표준 구조, 제목 키워드, 묶기/분리, 메뉴 경로 포맷)를 시스템 프롬프트로 강제합니다.

🔍유사 케이스 의미 검색

같은 태그 200건 중에서 LLM이 의미적으로 가까운 케이스만 골라 한 아티클로 묶을지 분리할지 판단합니다.

🔒내부 메모 식별

채널톡 options 배열의 private 플래그까지 파싱해 내부 대화와 고객 응대를 정확히 구분합니다.

✅자동 검증·보정

verify_draft가 헤딩 공백, 본문 길이, 종결형 비율, H1 누락 등을 체크하고 일부는 자동 보정합니다.

🎨채널톡 paste 호환

FAQ 답변은 마크다운 후처리로 코드블록 인식 문제를 차단. 메뉴 경로는 백틱 없이 화살표 한 줄로 통일합니다.

📊매니저 정성 분석

매니저별 반복 표현, 완충 표현, 종결형 분포까지 통계로 추출. 톤 일관성을 데이터로 점검합니다.
