🤖 ALF가 뭐예요?

ALF는 단순한 챗봇을 넘어 "실제 업무까지 처리하는 AI 에이전트"예요. 2025-11-28부터 v1/v2 통합되어 하나의 ALF로 운영됩니다.

핵심 동작 원리 — RAG

"고객 ALF는 검색 기반 생성(RAG) 기술을 활용해 답변 전 필요한 정보를 내부 지식에서 검색한 후 그에 기반한 답변을 생성합니다."

→ 즉, 잘 정리된 지식이 많을수록 답변 품질이 좋아져요.

📌 사용자가 채널톡에서 직접 해야 하는 것

  • 생성된 초안을 채널톡 도큐먼트로 옮기기
  • 폴더 구조 설계 + 폴더 참조 설명 작성
  • 퍼블리시 상태 변경
  • 필요 시 이미지 첨부 위치 확인 + 캡션 검토
  • 병합 분석 결과 검토 후 실제 병합 진행

📚 지식 구조 — 폴더와 항목

지식은 폴더(최대 5단계 계층) 안에 지식 항목을 담는 구조예요.

폴더 설정 항목

  • 이름: 폴더 구분용
  • 참조 설명: "이 폴더의 지식을 언제 참조해야 하는지" 기술 → ALF의 판단 기준
  • 필터링: 고객 정보·상담 정보 기반 조건 (특정 고객군에게만 지식 제공)
  • 동작 서비스: 채널톡, 카카오 상담톡, 네이버 톡톡, 전화, 인스타그램, 라인 중 선택

지식 항목 4가지 형식

  • 도큐먼트 (아티클/FAQ) — 공개 상태만 자동 추가, 스페이스 단위 ON/OFF, 편집은 도큐먼트 탭에서만
  • 웹사이트 — 최대 3개, 하위 페이지 최대 10,000개 자동 크롤링 (최대 12시간 소요)
  • 엑셀 — 10MB 이하, 참조 설명 필수, 한 시트당 하나의 표
  • PDF — 10MB 이하, 최대 500개, 텍스트만 인식 (이미지/표/레이아웃 미지원)

💬 FAQ — 단순·반복 질문에 효과적

형식 규정 (채널톡 공식)

  • 질문: 최대 100자
  • 추가 질문: 최대 100자, 최대 10개
  • 답변: 최대 500자
  • 등록 개수 제한 없음

ALF 동작 방식

"등록된 FAQ의 질문과 관련도가 높은 문의가 인입될 경우, FAQ의 답변을 참고하여 대답합니다."

→ 공개(published) 상태의 FAQ만 참조됩니다.

효과적인 작성 팁

"질문에는 정확한 키워드를 입력해 주시고, 추가 질문은 질문과 키워드를 포함해 다양하게 설정해 주세요."
"답변을 구체적으로 입력해 주셔야, 추후 ALF가 더 잘 활용할 수 있습니다."
  • 고객이 실제로 검색할 표현으로 질문 작성
  • 추가 질문엔 줄임말·구어체·오타까지 (예: "환불 어케해요?")
  • 답변은 결론부터, 조건 분기는 명확히

📄 아티클 — 복잡·다단계 시나리오에 효과적

아티클이란?

"아티클은 작성자가 작성한 글로 하나의 문서 단위"이며, 여러 블록을 조합해 본문을 구성해요.

사용 가능한 블록

  • 텍스트 / 제목 1~3단계 / 구분점·번호 목록 / 인용 / 콜아웃 / 코드블록 / 디바이더 / 표
  • 미디어: 이미지(32MB), 동영상(YouTube/파일), 파일, 웹페이지 북마크, 임베드
  • 마크다운 호환 (/로 커맨드 메뉴), 노션 복붙 가능

FAQ 통합

아티클 본문 안에 FAQ를 첨부할 수 있어요. 큰 가이드 + 자주 묻는 짧은 질문 조합으로 활용하면 효과적이에요. 비공개 FAQ는 고객에게 노출되지 않아요.

효과적인 구조

  • 제목: 고객이 검색할 표현 — # 구독 중인 플랜을 환불받는 방법
  • 첫 문단 = 결론: ALF는 앞부분을 우선 참조
  • 섹션 분리: ## 헤딩으로 케이스/단계
  • 조건 분기 명시: "경우에 따라" 금지, 케이스별로
  • 시각 요소: 비교는 표, 강조는 콜아웃(>), 절차는 번호 목록
  • 2,000자 이내 권장 (초과 시 분리)

SEO 설정

  • 대표 이미지: 700×366px 이상, 1.91:1 비율 권장
  • OG 태그 (이미지/제목/설명) 커스터마이징 가능
  • 클린 URL: 3~30자, 알파벳 소문자·숫자·하이픈·언더스코어

피해야 할 표현

  • ❌ "보통", "일반적으로", "경우에 따라"
  • ❌ "담당자에게 문의하세요" 단독 사용 → 조건/방법 함께 명시

⚙️ 규칙 (Rules) — ALF 비즈니스 맞춤화

규칙은 "ALF가 비즈니스 상황에 맞게 상담을 진행하도록 미리 작성하는 지시문"이에요. 기존의 역할 설명·맞춤 설명·용어사전이 통합됐어요.

  • 채널당 최대 30개, 규칙당 2,000자 이하
  • 필터링 조건으로 상황별 적용 가능
  • 개인화 변수 (고객 정보 자동 삽입)
  • 상담 처리 액션 (팀 배정, 태그 설정 등) 가능
  • 미리보기로 테스트 후 적용

규칙으로 해결할 수 있는 5가지 상황

상황문제해결 규칙 예시
전화 연결 제안 ALF가 실제 불가능한 통화 연결을 권유 "전화 연결을 권유하지 말고, 현재 대화 내에서 해결 가능한 방식으로 안내해주세요."
무분별한 약속 ALF가 "처리해드리겠습니다" 같은 약속 "업무 처리를 약속하는 표현 대신, 고객이 스스로 진행할 절차와 정보를 안내해주세요."
기능 혼동 A 기능 문의에 B 기능 정보까지 섞임 "A 기능 문의는 B, C, D 기능에 대해 안내하지 않고 A에 대해서만 안내해주세요."
상담원 연결 오류 "연결해드릴게요"라고만 하고 실제로 연결 안 됨 상담 열기 케이스만 규칙에 작성하고, 멘트는 제거하세요.
불필요한 추가 제안 고객 요청 외 정보까지 제공 "필요 이상의 추가 제안 없이, 핵심 정보만 명확하게 안내해주세요."

실전 규칙 예시 (복사해서 활용)

🔀 구분 규칙 — 기능 혼동 방지

"플랜 다운그레이드 문의는 '플랜 변경 가이드' 아티클만 참조하세요. '환불 안내' 아티클의 내용은 함께 안내하지 말 것."

🎙 톤 규칙 — 일관된 어투

"고객님 호칭을 사용하고, '~드립니다/안내드려요' 어투로 답변하세요. 친근하지만 거리감 있는 존대를 유지하세요."

📚 용어사전 규칙

"용어 정의: - 프리오더: 출시 전 상품 예약 주문 - 라이브클래스: 실시간 진행되는 강의 - 패키지: 여러 클래스를 묶어서 판매하는 상품"

⭐ 우선순위 규칙

"신규 가입 고객(가입 7일 이내) 문의는 '입문 가이드' 폴더의 아티클을 먼저 참조하세요."

🚫 제한 규칙 — 거짓 정보 방지

"확실하지 않은 환불 처리 기간, 결제 금액, 배송일정은 임의로 답변하지 마세요. 정확한 정보가 없으면 상담원 연결을 안내하세요."

  • ❌ "확실하지 않아도 최대한 답변해줘" — 부정확한 답변 유도
  • ✅ "불확실하면 상담 열기가 필요한지 먼저 물어봐" — 안전한 가드

⚡️ 태스크 (Task) — 실제 업무 자동화

태스크는 단순 Q&A를 넘어 "실제 업무를 자동으로 처리"하는 기능이에요. 주문 취소, 환불, 예약 변경 등을 ALF가 직접 처리해요.

태스크 도입이 좋은 경우

  • 주문 취소 / 배송지 변경 / 환불 처리
  • 예약 변경·취소, 구독 일정 변경
  • 정기 결제 취소, 서류 발급 요청

→ 반복 업무는 태스크로, 정보 제공은 FAQ/아티클로 분담해서 운영하면 효율적이에요.

채널톡 CS팀에 직접 문의해 받은 답변 3건을 원문 그대로 정리했어요. 공식 문서와 함께 참고하세요.

1️⃣ 효과적인 아티클 구조 가이드

알프가 더 잘 답변할 수 있도록 설정 시 참고할 내용

① 명확한 계층 구조 설계

  • 카테고리를 활용해 주제별로 지식을 분류 (예: '결제' 카테고리 안에 '환불 정책', '결제 수단' 아티클)
  • 성격이 다른 지식은 스페이스 단위로 구분하는 것이 효율적

② 구체적인 제목과 소제목 사용

  • 제목에는 핵심 키워드 포함
  • 본문 내에서도 소제목을 활용해 정보를 구조화 → ALF가 맥락을 파악하는 데 큰 도움

③ 데이터 최신화

  • 정확한 용어 사용
  • 관련 링크를 본문에 포함 → ALF가 고객에게 링크까지 함께 안내 가능
"짧은 문답. 답변이 고정적으로 정해진 내용은 FAQ 형식(최대 500자)이 유리하며, 복잡한 프로세스나 정책은 아티클로 상세히 작성해 주시면 좋을 것 같습니다."

④ 중복 지식 참조 관련

"동일한 내용이 도큐먼트와 웹사이트에 중복으로 존재할 경우, 답변 속도 자체에 직접적인 영향을 주기보다는 답변의 일관성과 정확도에 영향을 줄 수 있습니다."
  • 알프는 참조하는 지식의 양이 많아질수록 검토할 데이터가 늘어남 → 중복된 정보는 하나로 정리
  • 특히 내용이 미세하게 다를 경우 알프가 혼란을 느낄 수 있음 → 사이트 정보와 추가한 지식 내용이 다른 것은 없는지 꼼꼼히 체크

2️⃣ ALF가 찾기 어려운 문서 형태 5가지

① 제목/소제목 없이 본문에만 키워드가 있는 경우

"ALF는 제목과 소제목을 기준으로 우선 검색해요. 본문에만 내용이 있으면 찾지 못할 수 있어요."

② 표현이 모호하거나 중복된 내용이 여러 문서에 있는 경우

"비슷한 문장이 많으면 ALF가 잘못된 문서를 참조할 수 있어요."

③ 수정 중(미퍼블리시) 상태인 아티클

"퍼블리시 전 수정 중인 내용은 참조되지 않아요."

④ 이미지에만 정보가 담긴 경우

"ALF는 이미지 자체가 아닌 캡션(설명)을 참조해요. 캡션을 상세히 작성해야 정확도가 올라가요."

⑤ 폴더 안에 있는 지식인데 폴더 참조 설명이 질문과 맞지 않는 경우

"ALF가 폴더 자체에 진입하지 않아 내부 아티클을 탐색하지 못해요."

3️⃣ 아티클 작성 팁 (구조·문체)

"ALF는 문서의 구조를 파악할 때 제목과 소제목을 주요 기준으로 삼기 때문에, 구조를 명확히 나누는 것이 중요합니다."
  • 아티클 제목에는 핵심 키워드(상품명 등) 포함
  • 본문 내에서는 ## 또는 ###와 같은 소제목을 사용하여 섹션을 명확히 구분
"이미지만으로 구성된 아티클은 ALF가 내용을 참조하기 어렵습니다. 이미지 위아래에 내용을 설명하는 텍스트를 반드시 추가해 주세요."
"불렛 포인트(구분점 목록)나 번호 목록을 사용하면 정보의 선후 관계를 ALF가 훨씬 쉽게 파악합니다."
"또한 지식 베이스에 특정 어미에 대한 강제 규정은 없지만, 문맥이 명확한 종결형(-어요, -습니다)으로 작성하는 것을 권장드리고 있어요. 문장이 끊기지 않고 완성된 형태로 제공될 때 ALF가 정보의 의미를 가장 정확하게 해석할 수 있기 때문입니다."

⚠️ ALF가 찾기 어려운 문서 형태

채널톡 공식 안내. 아래 패턴은 피해주세요.

문제 상황왜 못 찾는지
제목/소제목 없이 본문에만 키워드 ALF는 제목·소제목 우선 검색. 본문만 있으면 매칭 실패
비슷한 표현·중복 내용이 여러 문서에 산재 ALF가 엉뚱한 문서를 참조할 수 있음
수정 중(미퍼블리시) 상태 아티클 퍼블리시 전 문서는 참조 안 됨
이미지에만 정보가 담긴 경우 ALF는 이미지가 아닌 캡션을 참조. 캡션 상세 작성 필수
폴더 참조 설명과 안 맞는 내용 ALF가 폴더 자체에 진입하지 않아 내부 아티클 탐색 안 함

✅ ALF 셋팅 체크리스트

  1. 지식 구조 설계: 폴더 계층(최대 5단계) 잡고, 각 폴더에 참조 설명 명확히
  2. FAQ 작성: 자주 묻는 짧은 질문 → 변형 질문 5~10개 포함
  3. 아티클 작성: 복잡한 시나리오 → 결론부터, 조건 분기 명시
  4. 비참조 항목 인식: 고객/상담 태그는 ALF가 안 봐요 → 본문에 정보 포함
  5. 필터링 설정: 특정 고객군용 지식은 조건 걸어두기
  6. 규칙 정의: 비즈니스 톤·정책·금지 표현은 규칙으로 가드
  7. 태스크 활용: 반복 업무는 태스크로 자동화
  8. 정기 업데이트: 웹사이트 변경 시 2시간 이상 간격으로 "다시 가져오기"
