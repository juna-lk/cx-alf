채널톡 공식 문서 기반으로 ALF를 효과적으로 활용하는 방법을 정리했어요.

출처: ALF란? · 지식 · FAQ · 아티클 · 규칙 · 태스크

🤖 ALF가 뭐예요?

ALF는 단순한 챗봇을 넘어 "실제 업무까지 처리하는 AI 에이전트"예요. 2025-11-28부터 v1/v2 통합되어 하나의 ALF로 운영됩니다.

핵심 동작 원리 — RAG

"고객 ALF는 검색 기반 생성(RAG) 기술을 활용해 답변 전 필요한 정보를 내부 지식에서 검색한 후 그에 기반한 답변을 생성합니다."

→ 즉, 잘 정리된 지식이 많을수록 답변 품질이 좋아져요.

📚 지식 구조 — 폴더와 항목

지식은 폴더(최대 5단계 계층) 안에 지식 항목을 담는 구조예요.

폴더 설정 항목

  • 이름: 폴더 구분용
  • 참조 설명: "이 폴더의 지식을 언제 참조해야 하는지" 기술 → ALF의 판단 기준
  • 필터링: 고객 정보·상담 정보 기반 조건 (특정 고객군에게만 지식 제공)
  • 동작 서비스: 채널톡, 카카오 상담톡, 네이버 톡톡, 전화, 인스타그램, 라인 중 선택

지식 항목 4가지 형식

  • 도큐먼트 (아티클/FAQ) — 공개 상태만 자동 추가, 스페이스 단위 ON/OFF, 편집은 도큐먼트 탭에서만
  • 웹사이트 — 최대 3개, 하위 페이지 최대 10,000개 자동 크롤링 (최대 12시간 소요)
  • 엑셀 — 10MB 이하, 참조 설명 필수, 한 시트당 하나의 표
  • PDF — 10MB 이하, 최대 500개, 텍스트만 인식 (이미지/표/레이아웃 미지원)

💬 FAQ — 단순·반복 질문에 효과적

형식 규정 (채널톡 공식)

  • 질문: 최대 100자
  • 추가 질문: 최대 100자, 최대 10개
  • 답변: 최대 500자
  • 등록 개수 제한 없음

ALF 동작 방식

"등록된 FAQ의 질문과 관련도가 높은 문의가 인입될 경우, FAQ의 답변을 참고하여 대답합니다."

→ 공개(published) 상태의 FAQ만 참조됩니다.

효과적인 작성 팁

"질문에는 정확한 키워드를 입력해 주시고, 추가 질문은 질문과 키워드를 포함해 다양하게 설정해 주세요."
"답변을 구체적으로 입력해 주셔야, 추후 ALF가 더 잘 활용할 수 있습니다."
  • 고객이 실제로 검색할 표현으로 질문 작성
  • 추가 질문엔 줄임말·구어체·오타까지 (예: "환불 어케해요?")
  • 답변은 결론부터, 조건 분기는 명확히

📄 아티클 — 복잡·다단계 시나리오에 효과적

아티클이란?

"아티클은 작성자가 작성한 글로 하나의 문서 단위"이며, 여러 블록을 조합해 본문을 구성해요.

사용 가능한 블록

  • 텍스트 / 제목 1~3단계 / 구분점·번호 목록 / 인용 / 콜아웃 / 코드블록 / 디바이더 / 표
  • 미디어: 이미지(32MB), 동영상(YouTube/파일), 파일, 웹페이지 북마크, 임베드
  • 마크다운 호환 (/로 커맨드 메뉴), 노션 복붙 가능

FAQ 통합

아티클 본문 안에 FAQ를 첨부할 수 있어요. 큰 가이드 + 자주 묻는 짧은 질문 조합으로 활용하면 효과적이에요. 비공개 FAQ는 고객에게 노출되지 않아요.

효과적인 구조

  • 제목: 고객이 검색할 표현 — # 구독 중인 플랜을 환불받는 방법
  • 첫 문단 = 결론: ALF는 앞부분을 우선 참조
  • 섹션 분리: ## 헤딩으로 케이스/단계
  • 조건 분기 명시: "경우에 따라" 금지, 케이스별로
  • 시각 요소: 비교는 표, 강조는 콜아웃(>), 절차는 번호 목록
  • 2,000자 이내 권장 (초과 시 분리)

SEO 설정

  • 대표 이미지: 700×366px 이상, 1.91:1 비율 권장
  • OG 태그 (이미지/제목/설명) 커스터마이징 가능
  • 클린 URL: 3~30자, 알파벳 소문자·숫자·하이픈·언더스코어

피해야 할 표현

  • ❌ "보통", "일반적으로", "경우에 따라"
  • ❌ "담당자에게 문의하세요" 단독 사용 → 조건/방법 함께 명시

⚙️ 규칙 (Rules) — ALF 비즈니스 맞춤화

규칙은 "ALF가 비즈니스 상황에 맞게 상담을 진행하도록 미리 작성하는 지시문"이에요. 기존의 역할 설명·맞춤 설명·용어사전이 통합됐어요.

  • 채널당 최대 30개, 규칙당 2,000자 이하
  • 필터링 조건으로 상황별 적용 가능
  • 개인화 변수 (고객 정보 자동 삽입)
  • 상담 처리 액션 (팀 배정, 태그 설정 등) 가능
  • 미리보기로 테스트 후 적용

⚡️ 태스크 (Task) — 실제 업무 자동화

태스크는 단순 Q&A를 넘어 "실제 업무를 자동으로 처리"하는 기능이에요. 주문 취소, 환불, 예약 변경 등을 ALF가 직접 처리해요.

태스크 도입이 좋은 경우

  • 주문 취소 / 배송지 변경 / 환불 처리
  • 예약 변경·취소, 구독 일정 변경
  • 정기 결제 취소, 서류 발급 요청

→ 반복 업무는 태스크로, 정보 제공은 FAQ/아티클로 분담해서 운영하면 효율적이에요.

✅ ALF 셋팅 체크리스트

  1. 지식 구조 설계: 폴더 계층(최대 5단계) 잡고, 각 폴더에 참조 설명 명확히
  2. FAQ 작성: 자주 묻는 짧은 질문 → 변형 질문 5~10개 포함
  3. 아티클 작성: 복잡한 시나리오 → 결론부터, 조건 분기 명시
  4. 비참조 항목 인식: 고객/상담 태그는 ALF가 안 봐요 → 본문에 정보 포함
  5. 필터링 설정: 특정 고객군용 지식은 조건 걸어두기
  6. 규칙 정의: 비즈니스 톤·정책·금지 표현은 규칙으로 가드
  7. 태스크 활용: 반복 업무는 태스크로 자동화
  8. 정기 업데이트: 웹사이트 변경 시 2시간 이상 간격으로 "다시 가져오기"
